请在 experiment2_force_maintenance.py 中补充“固定初始位姿 + INIT 阶段手动开始”的机制，用于减少每轮实验开始前机械臂对位差异和碰撞风险。

背景：
实验二需要比较 visual_only 和 vision_haptic 两种条件下，受试者能否维持 PyBullet 指尖接触力在目标范围内。为了保证两种条件可比，每轮正式 trial 开始前，机械臂和夹爪应尽量处于一致的初始位姿。INIT 阶段不计入正式实验时间，受试者/实验者可以微调，确认夹爪和物体相对位置合适后按 S 手动开始正式 trial。

========================
一、配置文件新增参数
========================

请在 experiment2_config.yaml 中加入或扩展以下配置：

experiment:
  initial_settle_steps: 240        # reset 后预移动到 initial_action 的仿真步数
  auto_move_to_initial: true       # 是否 reset 后自动移动到初始位姿
  manual_start_enabled: true       # 是否允许按 S 手动开始
  trial_duration_sec: 120.0
  init_timeout_sec: 60.0

initial_action:
  x: 0.0
  y: 0.0
  z: 0.5
  roll: 0.0
  pitch: 1.57079632679
  yaw: 1.57079632679
  gripper_opening: 0.04

logging:
  log_init_phase: false            # 默认不记录 INIT_PREPARE 的 time-series，避免污染正式数据

说明：
- initial_action 表示实验开始前的末端目标位姿和夹爪开口，不是机器人底座位置。
- 不要修改 UR5Robotiq85((0, 0.5, 0), (0, 0, 0)) 这样的机器人底座参数。
- 正式实验优先使用固定 initial_action + S 手动开始。
- 只保留 S 手动开始正式 trial，不再使用 TARGET_RANGE_STABLE 自动开始。

========================
二、reset 后预移动到 initial_action
========================

创建函数：

def build_action_from_config(cfg) -> np.ndarray:
    从 cfg["initial_action"] 读取 x, y, z, roll, pitch, yaw, gripper_opening，
    返回 np.array([x, y, z, roll, pitch, yaw, gripper_opening], dtype=float)。

创建函数：

def move_to_initial_pose(env, action, steps, sim_dt=1.0/240.0):
    """
    在正式实验开始前，将机器人移动到配置的初始末端位姿。
    该阶段不计入 trial_elapsed_sec。
    """
    for _ in range(steps):
        if not p.isConnected():
            break
        env.robot.move_ee(action[:6], control_method="end")
        env.robot.move_gripper(action[6])
        env.step_simulation()

主程序初始化流程改为：

1. 创建环境 env。
2. env.reset()。
3. 从 yaml 读取 initial_action，构造 action。
4. 如果 experiment.auto_move_to_initial 为 true：
   调用 move_to_initial_pose(env, action, initial_settle_steps)。
5. 进入 INIT 状态。
6. INIT 阶段允许用户继续用键盘微调 x/y/z/rpy/gripper_opening。
7. 只有触发 MANUAL_START 后才开始正式 trial 计时。

注意：
- move_to_initial_pose 阶段不要启动 AUTO_PERTURB_CLOSE / AUTO_PERTURB_OPEN。
- move_to_initial_pose 阶段不要发送正式触觉反馈；可以发送 0,0 或保持关闭。
- move_to_initial_pose 阶段不要计入正式 time_series，除非 logging.log_init_phase=true。
- move_to_initial_pose 只做预定位，不保证接触力已经在目标范围内。
- 如果 experiment.auto_move_to_initial=false，则直接进入 INIT，并将 initial_pose_reached 记为 true。

========================
三、INIT 阶段微调与手动开始
========================

INIT 阶段：
- 允许用户使用原有键盘控制微调机械臂末端和夹爪。
- I/K, J/L, U/O 控制 x/y/z。
- 1/2, 3/4, 5/6 控制 roll/pitch/yaw。
- Z/X 控制 gripper_opening。
- S 用于手动开始正式 trial。
- Q 退出。

按 S 的行为：
如果 state == INIT 且 manual_start_enabled == true：
    记录 MANUAL_START 事件；
    manual_start = true；
    trial_start_sim_time = sim_time；
    trial_elapsed_sec = 0；
    state = AUTO_WAIT；
    生成下一次扰动等待时间；
    正式 trial 开始。

MANUAL_START 事件写入 event CSV：
event_type = "MANUAL_START"
detail = "manual start from INIT after operator confirmed initial pose"

优先级：
- S 在整个 INIT 阶段始终有效。
- 正式实验中 S 不再用于开始，避免重复触发。
- S 只在 INIT 状态有效。
- init_timeout_sec 只负责提醒，不负责解锁或自动开始。

========================
四、时间和日志
========================

时间定义：
- timestamp_iso 使用真实墙钟时间。
- sim_time 使用固定仿真步长累计，每次主循环 sim_time += 1/240。
- time_sec 写 sim_time。
- trial_elapsed_sec = sim_time - trial_start_sim_time。
- INIT 阶段 trial_elapsed_sec 固定写为 -1。
- 正式 trial 开始后 trial_elapsed_sec 从 0 开始。

time_series CSV 新增字段：
manual_start
trial_elapsed_sec
init_phase
initial_pose_reached

字段含义：
- manual_start: 本轮 trial 是否由 S 手动开始。
- trial_elapsed_sec: 正式 trial 开始后的仿真累计时间。
- init_phase: 当前是否仍处于 INIT 或 INIT_PREPARE。
- initial_pose_reached: 是否已经执行完 move_to_initial_pose。

如果 logging.log_init_phase=false：
- INIT_PREPARE 阶段不写 time_series。
- INIT 微调阶段可以不写 time_series，或只写 event。
- 正式 trial 开始后才写用于分析的 time_series。

如果 logging.log_init_phase=true：
- 可以记录 INIT 阶段，但必须通过 init_phase 字段标记，后续分析默认排除。

event CSV 需要包含：
EXPERIMENT_START
INITIAL_POSE_START
INITIAL_POSE_DONE
MANUAL_START
AUTO_PERTURB_START_CLOSE
AUTO_PERTURB_START_OPEN
AUTO_PERTURB_END
USER_INPUT_Z
USER_INPUT_X
FORCE_ABOVE_HIGH
FORCE_BELOW_LOW
FORCE_BACK_IN_RANGE
EMERGENCY_HIGH_FORCE
EXPERIMENT_END

事件语义：
- INITIAL_POSE_START：开始执行 move_to_initial_pose。
- INITIAL_POSE_DONE：完成 initial_settle_steps 后进入 INIT。
- MANUAL_START：实验者按 S，正式 trial 开始。
- 所有 FORCE_ABOVE_HIGH / FORCE_BELOW_LOW / FORCE_BACK_IN_RANGE 仍然按边沿触发。

========================
五、正式 trial 结束条件
========================

正式 trial 从 MANUAL_START 之后开始计时。

if trial_elapsed_sec >= trial_duration_sec:
    记录 EXPERIMENT_END；
    发送 ESP32 0,0；
    保存 CSV；
    正常退出。

INIT 阶段不参与 trial_duration_sec 计时。

如果 INIT 持续超过 init_timeout_sec：
    命令行提示：
    "INIT timeout. Press S to start manually after confirming pose, or Q to quit."
不要自动强制开始。

========================
六、调试辅助：保存当前 action
========================

请不要在正式 experiment2 中占用 S 以外的开始逻辑。

如果方便，请新建一个单独调试脚本或在 forcelevel_simulation.py 中增加一个按键功能：
- 按 V 打印当前 action 为 yaml 格式：
  initial_action:
    x: ...
    y: ...
    z: ...
    roll: ...
    pitch: ...
    yaw: ...
    gripper_opening: ...

这个功能用于实验前手动调试初始位姿，然后把打印结果复制到 experiment2_config.yaml。

不要在 experiment2_force_maintenance.py 中用 V 改配置文件，只打印即可，避免误操作。

========================
七、安全限制
========================

所有 action 都必须经过 yaml 中 limits 限制：
- x/y/z/rpy 根据已有控制范围限制；
- gripper_opening 使用 grip_min/grip_max；
- 自动扰动时额外使用 safe_grip_min/safe_grip_max。

INIT 阶段用户微调也要 obey limits，避免机械臂大幅移动撞到环境。

如果 p.isConnected() 为 false，主循环应正常退出，不要报 Not connected to physics server。

finally 中必须：
- ESP32 发送 0,0；
- 关闭 socket；
- 关闭 CSV 文件；
- disconnect PyBullet。

========================
八、实现目标
========================

最终效果：
1. 程序启动后 reset 环境。
2. 机械臂自动移动到 yaml 中 initial_action 附近。
3. 进入 INIT，用户可以手动微调。
4. 实验者确认夹爪和物体位置合适后按 S。
5. 记录 MANUAL_START，trial_elapsed_sec 从 0 开始。
6. 状态进入 AUTO_WAIT。
7. 系统按实验二逻辑随机自动夹紧/松开夹爪目标开口。
8. 用户根据视觉/触觉反馈按 Z/X 维持接触力。
9. trial_duration_sec 到达后自动结束并保存数据。
