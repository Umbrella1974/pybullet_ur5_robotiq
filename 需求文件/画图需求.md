========================
三、指标定义
========================

1. contact_force

定义：
contact_force = tf_force_smooth + if_force_smooth

含义：
左右指尖平滑接触法向力之和，表示仿真夹持接触强度。

2. force_error

定义：
target_mid = (target_low + target_high) / 2
force_error = abs(contact_force - target_mid)

含义：
当前接触力与目标力中心的偏差。越小越好。

3. in_target_range_ratio

定义：
正式 trial 阶段中，contact_force 位于 [target_low, target_high] 内的采样点比例。

计算：
in_target_range_ratio = mean(in_target_range)

含义：
比例越高，说明越能稳定维持目标夹持力。

4. disturbance_response_time

这是本项目优先使用的响应时间指标，不要把它叫成人体纯反应时间。

定义：
从系统自动扰动开始，到用户第一次做出正确方向输入的时间。

规则：
- AUTO_PERTURB_CLOSE 表示系统自动夹紧，接触力倾向变大；
  正确用户反应是“张开夹爪”的输入。
- AUTO_PERTURB_OPEN 表示系统自动松开，接触力倾向变小；
  正确用户反应是“闭合夹爪”的输入。

计算：
disturbance_response_time = t_first_correct_user_input - t_auto_perturb_start

优点：
它不依赖 target_low / target_high 的人为阈值，因此比基于越界的 reaction_time 更稳。

注意：
由于不同代码版本中用户开合按键字段可能不同，请用以下策略：
- 优先根据 event CSV 中 USER_INPUT_OPEN / USER_INPUT_CLOSE / USER_INPUT_Z / USER_INPUT_X 等事件判断；
- 如果没有明确 open/close 事件，就根据 time-series 中 gripper_opening 的变化方向判断：
  - gripper_opening 增大 = 用户张开夹爪；
  - gripper_opening 减小 = 用户闭合夹爪；
- 如果无法判断正确方向，则该次扰动的 response_time 记为 NaN，不要强行计算。

5. recovery_time

定义：
从自动扰动开始，到 contact_force 回到 [target_low, target_high] 并连续保持 recovery_hold_sec 的时间。

默认：
recovery_hold_sec = 0.2 秒

计算：
- 起点：AUTO_PERTURB_START_CLOSE 或 AUTO_PERTURB_START_OPEN 的时间；
- 终点：此后第一次出现 contact_force 连续 recovery_hold_sec 都在目标范围内的时刻；
- recovery_time = 终点 - 起点；
- 如果直到下一次扰动或 trial 结束都没有恢复，则记为 NaN 或 censored，不要填 0。

注意：
recovery_time 仍然依赖 target_low / target_high，因此论文里应说明 target_low/high 来自预实验标定，并在所有条件下保持一致。

6. 可选：time_above_high_ratio / time_below_low_ratio

定义：
time_above_high_ratio = mean(contact_force > target_high)
time_below_low_ratio = mean(contact_force < target_low)

可以用于画目标区间内/高于区间/低于区间的时间占比图。

========================
六、关于 reaction_time 的处理
========================

不要把基于 target_low / target_high 越界的 reaction_time 作为主图。

原因：
target_low / target_high 是实验人为设定的目标范围，小样本本科实验中不适合把“越界后反应时间”作为唯一核心证据。

可以保留一个可选分析：
threshold_reaction_time

定义：
- 当 contact_force 从目标范围内越过 target_high 后，到第一次正确张开输入的时间；
- 当 contact_force 从目标范围内低于 target_low 后，到第一次正确闭合输入的时间。

但是：
- 默认不作为主图；
- 只在用户显式开启参数 --include-threshold-reaction 时计算；
- 图名不要叫 pure reaction time，叫 closed-loop response time。

主图优先使用：
disturbance_response_time

========================
七、小样本绘图原则
========================

由于实验可能只有 1~2 位受试者：
1. 不做显著性检验。
2. 不画带显著性星号的图。
3. 不优先画单纯柱状图。
4. 优先画：
   - 单 trial 过程图；
   - 每个 trial 的散点；
   - 同一 participant 的配对线；
   - 分布点图。
5. 结果表述用“趋势”“重复试次中表现出改善”“具有一定辅助作用”，不要写“显著提高”。

十、论文解释口径
========================

请在代码注释或 README 中说明：

1. mean_force_error 和 in_target_range_ratio 是主要稳定性指标。
2. disturbance_response_time 是主要响应速度指标，它从自动扰动开始计时，不依赖 target_low/high 的越界判定。

4. threshold_reaction_time 仅作为可选分析，不作为主指标。
5. 本实验是小样本、重复试次的可行性验证，不进行显著性检验。
