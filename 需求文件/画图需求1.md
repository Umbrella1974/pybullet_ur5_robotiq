同意，按以下口径实现：

1. event CSV 新版本增加 trial_elapsed_sec 字段；绘图脚本兼容旧数据。
   如果旧 event CSV 没有 trial_elapsed_sec，则用 MANUAL_START 或 TARGET_RANGE_STABLE 的 time_sec 作为 trial_start_time_sec，
   event_trial_elapsed_sec = event_time_sec - trial_start_time_sec。

2. trial_id 使用 participant_id + condition + 顺序编号：
   例如 P01_visual_only_T01、P01_vision_haptic_T01。
   原始文件名或时间戳保留为 source_file 字段，不作为主要 trial_id。

3. 一次扰动没有等到正确方向用户输入时：
   disturbance_response_time = NaN。
   summary_by_trial 中额外输出 n_response_events、n_valid_response_events、response_valid_ratio。

4. 一次扰动到下一次扰动或 trial 结束都没有恢复到目标范围时：
   recovery_time = NaN。
   summary_by_trial 中额外输出 n_recovery_events、n_valid_recovery_events、recovery_valid_ratio。

5. 论文主图按 participant mean per condition 聚合。
   即先对每个 participant 在每个 condition 下的 trial 求平均，再画配对点/配对线。
   trial-level 图保留为补充图或 debug 图。

6. disturbance_response_time 和 recovery_time 的 condition 主图默认合并 AUTO_PERTURB_CLOSE 与 AUTO_PERTURB_OPEN。
   额外输出 close-only / open-only 补充图。

7. 默认导出 PNG。
   --paper-mode 时同时导出 SVG。

8. CLI 默认行为：
   不加参数时默认执行 --summary。
   --single FILE_OR_GLOB 只画指定 trial。
   后续可以加 --all-trials 画所有单 trial 图。
