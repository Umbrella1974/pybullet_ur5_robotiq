========================
四、状态机
========================

请实现清晰的状态机，至少包括：

INIT_PREPARE
INIT
USER_CONTROL
AUTO_WAIT
AUTO_PERTURB_CLOSE
AUTO_PERTURB_OPEN

状态含义：

INIT_PREPARE：
- reset 后如果 auto_move_to_initial=true，先自动移动到 initial_action 附近。
- 该阶段不计入正式 trial。
- 不启动自动扰动；ESP32 保持 0,0 或不发送正式反馈。
- 完成后进入 INIT。

INIT：
- 实验刚开始，先进入准备阶段。
- 用户手动调整机械臂末端和夹爪。
- 只有按下 S 才开始正式 trial，不再使用 TARGET_RANGE_STABLE 自动开始。
- S 在整个 INIT 阶段始终有效。
- 触发 MANUAL_START 时：
  - trial_start_time = now
  - trial_elapsed_sec 从 0 开始
  - state 从 INIT 进入 AUTO_WAIT
- 如果 INIT 超过 init_timeout_sec，仅打印提醒信息，不负责解锁或自动开始。

USER_CONTROL：
- 用户正在按 Z 或 X。
- 用户优先。
- 自动扰动清零或暂停。
- 用户松开按键后进入 AUTO_WAIT。

AUTO_WAIT：
- 用户刚停止输入。
- auto_wait_sec 是最小保护等待时间。
- perturb_interval_min_sec / perturb_interval_max_sec 表示从“用户停止输入或上次扰动结束”到“下一次扰动开始”的总等待时间。
- 实际等待时间 = max(auto_wait_sec, random interval)。
- 等待结束后，随机选择 AUTO_PERTURB_CLOSE 或 AUTO_PERTURB_OPEN。

AUTO_PERTURB_CLOSE：
- 自动轻微闭合夹爪。
- 使用 auto_grip_speed（单位 m/s），每帧扰动量 = auto_grip_speed * (1/240)。
- 目的是让 PyBullet 指尖接触力变大。
- 但必须满足安全条件：
  - contact_force < emergency_high；
  - gripper_opening > safe_grip_min；
  - allow_auto_close 为 true。
- 如果用户按键，立即进入 USER_CONTROL。
- 如果自动扰动持续超过 perturb_duration_sec，或累计扰动超过 max_auto_delta，停止扰动，进入 AUTO_WAIT。
- 如果 contact_force 超过 emergency_high，立即停止自动闭合，进入 AUTO_WAIT。

AUTO_PERTURB_OPEN：
- 自动轻微张开夹爪。
- 使用 auto_grip_speed（单位 m/s），每帧扰动量 = auto_grip_speed * (1/240)。
- 目的是让 PyBullet 指尖接触力变小，甚至接触减弱。
- 但必须满足安全条件：
  - gripper_opening < safe_grip_max；
  - allow_auto_open 为 true。
- 如果用户按键，立即进入 USER_CONTROL。
- 如果自动扰动持续超过 perturb_duration_sec，或累计扰动超过 max_auto_delta，停止扰动，进入 AUTO_WAIT。
- 如果连续 no_contact_frames_required 帧无接触，则停止自动松开，进入 AUTO_WAIT，避免物体完全丢失后继续松开太多。

重要：
- 自动扰动是实验扰动，不是控制器。
- 用户按键永远优先。
- 用户输入时不允许 AUTO_PERTURB_CLOSE 或 AUTO_PERTURB_OPEN 继续修改夹爪。
- 自动扰动中的 dt 使用固定仿真步长 1/240 秒，不使用 wall-clock delta。
- 每轮 active trial 固定运行 trial_duration_sec 后自动结束。

========================
五、接触力读取
========================

请复用或改进已有函数 read_separate_fingertip_forces(env)。

读取：
p.getContactPoints(bodyA=env.robot.id, bodyB=env.boxID)

只统计 robot_link_index == TF_LINK 或 IF_LINK 的 contact[9] normalForce。

返回：
tf_force_sum
tf_max_force
tf_contact_count
if_force_sum
if_max_force
if_contact_count

平滑：
tf_force_smooth = (1-alpha) * tf_force_smooth + alpha * tf_force_sum
if_force_smooth = (1-alpha) * if_force_smooth + alpha * if_force_sum

整体接触力：
如果 combine_method == "max":
    contact_force = max(tf_force_smooth, if_force_smooth)
如果 combine_method == "sum":
    contact_force = tf_force_smooth + if_force_smooth

实验二默认并推荐使用：
combine_method == "sum"
即用左右指尖平滑力之和作为整体接触力。

接触状态：
has_contact = contact_force > contact_threshold
但不要只看一帧。

稳定接触与稳定失联优先按秒判断：
- 使用 contact_hold_sec 和 no_contact_hold_sec 作为阈值；
- 如果 has_contact 当前为 true：
  - contact_hold_time_sec += 1/240
  - no_contact_hold_time_sec = 0
- 否则：
  - no_contact_hold_time_sec += 1/240
  - contact_hold_time_sec = 0

stable_contact = contact_hold_time_sec >= contact_hold_sec
stable_no_contact = no_contact_hold_time_sec >= no_contact_hold_sec

同时保留 frame-based 计数作为调试字段：
- 如果 has_contact 当前为 true，contact_frames += 1, no_contact_frames = 0
- 否则 no_contact_frames += 1, contact_frames = 0

========================
六、反馈等级
========================

force_to_level(force, thresholds):
- thresholds 来自 yaml 的 level_thresholds，例如 [5.0, 50.0, 100.0]
- force < thresholds[0] → level 0
- thresholds[0] <= force < thresholds[1] → level 1
- thresholds[1] <= force < thresholds[2] → level 2
- force >= thresholds[2] → level 3

如果 condition == visual_only：
    ESP32 发送 0, 0
如果 condition == vision_haptic：
    ESP32 发送 tf_level, if_level

发送频率：
- 不要每一帧疯狂发送。
- 使用 feedback_send_period_sec 作为最小发送间隔，限制正式反馈的最大发送频率。
- 如果 payload 在发送间隔内连续变化，保留最新目标值，并在下一次允许发送时发出。
- 如果 payload 没变，但超过 keepalive_send_period_sec，也发送一次，便于以后 ESP32 加 watchdog。
- keepalive 计时逻辑放在实验主循环中（记录上次发送时间），不修改已有的 ESP32FeedbackClient 类，保持与 forcelevel_simulation.py 和 experiment1_contact_judgment.py 的向后兼容。

程序退出时必须发送 0, 0，然后关闭 socket。
